# ADR-0002: 상태관리 선택

- 상태: Accepted
- 날짜: 2026-05-26
- 결정자: 주현빈

## 배경
Flutter 앱에서 화면 간 상태(여행 일정, 지도 데이터 등)를 어떻게 관리할지 결정이 필요하다.

## 고려한 대안

### 대안 A: Provider
- 장점: 공식 권장, 학습 곡선 낮음, 코드 단순
- 단점: 복잡한 상태에서 한계

### 대안 B: Riverpod
- 장점: Provider 개선판, 타입 안전, 테스트 용이
- 단점: Provider보다 학습 곡선 있음

### 대안 C: Bloc
- 장점: 대규모 앱에 적합, 명확한 구조
- 단점: 보일러플레이트 코드 많음, 초급자에게 복잡

## 결정
Provider를 선택한다.

## 이유
- 초급 수준에서 가장 빠르게 익힐 수 있음
- Flutter 공식 문서에서 권장
- TR 앱 규모에서 충분한 기능 제공

## 결과
- 긍정: 빠른 개발 가능, 레퍼런스 많음
- 부정: 앱이 커지면 Riverpod으로 전환 필요할 수 있음
