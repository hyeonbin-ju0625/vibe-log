# ADR-0003: 백엔드 선택

- 상태: Accepted
- 날짜: 2026-05-26
- 결정자: 주현빈

## 배경
TR 앱의 일정 저장 및 AI API 호출을 어떻게 처리할지 결정이 필요하다.

## 고려한 대안

### 대안 A: Firebase
- 장점: 빠른 설정, 실시간 DB, 인증 내장
- 단점: Google 의존성, 비용 발생 가능

### 대안 B: REST API (직접 구축)
- 장점: 완전한 제어권
- 단점: 서버 개발 필요, 시간 많이 걸림

### 대안 C: 로컬 저장 (SQLite)
- 장점: 서버 불필요, 오프라인 작동
- 단점: 기기 간 동기화 불가

## 결정
Firebase를 선택한다.

## 이유
- 별도 서버 개발 없이 빠르게 구축 가능
- 일정 저장(Firestore)과 인증 기능 바로 사용 가능
- Claude API, Google Maps API와 함께 사용하기 적합한 환경

## 결과
- 긍정: 빠른 개발, 확장성 있음
- 부정: Firebase 의존성 생김, 무료 한도 초과 시 비용 발생
